DHCP Relay

1. Créer 3 : vm 2 WIN10, une pfsense et une Ubuntu server qui servira de serveur dhcp

2. Configurer le pfsense

Login :

  • maxime
  • max

Plan IP :

Win10 DHCP
10.0.0.0 /16 192.168.10.0 /24
10.0.0.1 /16 192.168.10.1 /24

10.0.0.0 - 10.0.1.245

Range de 500pc

Mettre 2 cartes réseau au pfsense

Bien mettre les @IP sur le client W10 :

  • admin
  • pfsense

Nommer GW01 : pfsense 2 carte réseaux ici une "client" et une "réseau" tout en réseau interne

Serveur DHCP dans le réseau serveur

IP client 10.0.0.255.254/16

IP serveur 192.168.10.254 /24

Aller sur le Win 10 et aller dans le pfsense en ligne (http://192.168.10.254 / login : admin | passwd : pfsense)

Tester un ping sur la passerelle (bien dans le même réseau ? le même que dans la carte du pfsense ? bonne adresse dans le pfsense ?)

La machine de debug est dans le réseau serveur avec comme IP 192.168.10.10

Activer le DHCP Relay sur pfsense en ligne (dans service)

Sur l'interface client

  • Enable l'interface
  • Destination server vers 192.168.10.1 (IP du serveur)
  • journalctl -xe pour voir les logs sur Linux
  • Bien mettre le serveur DHCP dans le bon réseau interne
